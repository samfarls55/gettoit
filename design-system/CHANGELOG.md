# Design System Changelog

Append-only log of every change to the design system spec. Newest at top. One line per change unless a `BREAKING:` entry needs context.

## 2026-05-14

- **Quiz surfaces (S03–S07):** documented the **Question card cross-fade** in `motion.md` and paired the iOS content router (`ios/Sources/App/QuizScreen.swift`) to the existing `--grad-tween` token (1100ms ease-in-out). Previously the foreground question card swapped instantly while the background gradient was still running its 1100ms tween, which read as ~1s of motion lag on every Q-to-Q transition (founder dogfood). Strategy: tune card up to match gradient — the gradient duration is locked per `motion.md` §"Gradient surface tween" (load-bearing time-of-day read), so the card adopts the same duration + easing rather than the other way around. No new motion token — the card transition aliases `--grad-tween` so the pair stays ms-exact by construction. Reduced motion drops both layers to 300ms linear in lockstep. (bug-04 v1.1, #44.)
- Registered **`color.text.on-bright-gradient.secondary`** (`rgba(14,16,17,0.78)` — tinted ink). New semantic-roles row for surfaces whose gradient reaches the yellow/peach band (initiator, Q1, Q2 yellow-bottom; verdict, checkin yellow-top). White-on-yellow secondary subhead collapses to ~1.3:1 against `#FFD23F` and fails WCAG AA; tinted-ink-at-0.78 measures **7.74:1** there with ≥5.62:1 across the coral top of the initiator — clears AA body-text with 1.1+ points of headroom across the entire vertical extent. New role rather than tuning the existing `on-gradient.secondary` because the existing role still works on the indigo/midnight surfaces (Q3–Q5, waiting, midnight) and the swift generator's `Color.white.opacity(...)` template is architecturally aligned with white-tinted alpha — cascading the change would regress dark surfaces. New SwiftUI surface: `GTIColor.TextOnBrightGradient.secondary` (emitted by `gen-swift.mjs`). Initiator subhead, eyebrow, radius value, and vertical-row meta migrated; shared `QuizQuestionHeader` (renders on Q3–Q5 too) intentionally NOT migrated — Q1/Q2 subhead contrast is a separate follow-up. Updated `accessibility.md` §1.1 + new §1.1.1 contrast table. (sg-01 v1.1, issue #45.)
- Added **surface S00 Landing** (`surfaces/00-landing.md`, `code/screens/ScreenLanding.jsx`). Post-sign-in entry surface with exactly two CTAs — `"Start a Decision"` (routes to S01 Initiator) and `"Account Settings"` (routes to S09 Settings). No fields, no controls — the surface is a router, not a destination. Reuses the existing `initiator` gradient stop and existing `C-05` Primary Pill CTA (white + ghost variants). No new components, no new tokens. Visual / brand polish deferred to the pre-public-launch milestone; the structural skeleton ships now so routing + tap targets are real for the iOS wire ([[../gti-vault/15_issues/v1.1/issues/tb-01-landing-page-wire|TB-01]]). Adjacency flagged: S01's existing `"SETTINGS"` footer link becomes redundant once this surface routes to S09 as a first-class CTA; whether to remove the S01 footer is a separate spec decision not actioned here. (sg-02.)
- Added **surface S00a Forced Sign-in Gate** (`surfaces/00a-signin.md`, `code/screens/ScreenSignIn.jsx`). First-launch iOS gate — single Sign-in-with-Apple affordance, no skip / continue-as-guest path. Closes the iOS half of [[../gti-vault/60_engineering/adr/0007-auth-anonymous-default-apple-upgrade|ADR 0007]] for v1.1; web fallback remains anonymous-default. Reuses the `initiator` gradient stop for visual continuity into S01. No new components — the pill is the existing `PillCTA white` + the Apple-glyph prefix slot the C-22 `default` state already uses. Locked copy: eyebrow `"Tonight's session"`, headline `"Pick up where you left off"`, CTA `"Save my taste profile"`. (sg-03.)
- **S04 Waiting:** added the **"Download the app" CTA** for web-fallback anonymous voters. Reuses `PillCTA white` (no new component). Rendered only when `platform === 'web'` AND `isAnonymous`; iOS users (already in the app) and any Apple-linked web users render `hidden`. Exactly one of {C-22 chip, this CTA} ever renders in a given context. Sub-line `"Then your votes save with you"` (eyebrow token, white 0.6) sits below the pill. Locked copy: `"Download the app"`. Tap opens the App Store deep-link and emits a `waiting_download_cta_tapped` telemetry event. Web-to-iOS identity merge is NOT implemented — the CTA copy promises forward-going behavior only. (sg-03.)

- Added **C-23 LocationPicker** component (`components.md`, `code/components.jsx`) — reusable location selector per [[../gti-vault/60_engineering/adr/0009-locationpicker-as-reusable-component|ADR 0009]]. Two JSX exports compose one conceptual primitive: `LocationPickerChip` (persistent readout) + `LocationPickerSheet` (typeahead editor with `MKLocalSearchCompleter` suggestion list, "Use current location" affordance, deny-state Settings deep-link). - **S05 Verdict:** added `solo` mode for single-member rooms. Triggered when `members.length === 1` AND the initiator did NOT share an invite (the share sheet never opened). Eyebrow + hero + meta + time badge + rule chip + `"I'm in"` + reroll remain. **Suppressed:** voice-receipt row (one voice doesn't need to be receipted back to itself). **Replaced:** the `default` mode's save-group secondary is replaced with the C-22 Auth Upgrade Chip (`"Save this taste profile"`) — solo is the highest-conversion moment for Apple Sign-in. Time badge audience reads `"You"` rather than `"All N of you"`. `rule_text` follows the same generator (no member-count branch — the engine never references `votes.length`), so `"N of M wanted X"` is structurally impossible. (TB-13.)
- **S05 Verdict:** added pre-permission line below the CTA dock — `"We'll check in tomorrow — see if you went."` Voluntary warm-friend register; the line fires the native iOS push permission prompt after the first "I'm in" tap (once per session). Suppressed in `read-only` (no ratification path) and `no-survivor` (no verdict to check in on). `code/screens/ScreenVerdict.jsx` renders the line; the iOS port lives in `ios/Sources/App/VerdictScreen.swift`. (TB-08.)
