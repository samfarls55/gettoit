# Search area picker

Outcome of the 2026-06-03 `/grill-with-docs` session for replacing the text-first C-23 LocationPicker + separate distance slider with an Apple MapKit search-area control.

## Locked decisions

### Q1 â€” Combined concept

**Locked:** combine location and radius into one **Search area** control.

Setup should no longer ask users to reason about `Where to` and `How far` as separate tasks. The user-facing concept is a geographic area used to fetch candidate venues.

### Q2 â€” Timezone

**Locked:** Search area has no timezone or timing semantics.

Existing docs that tie the Plan reroll window to a "search-area timezone" are now flagged as stale/conflicting. Timing behavior needs a separate correction; the map selector must not reinforce the old model.

### Q3/Q4 â€” Map interaction model

**Locked:** mirror Airbnb-style map behavior.

The Search area editor is a full-screen Apple MapKit surface. The user pans and pinches the map; the visible map area drives the selected area. The committed representation remains center coordinate plus radius:

- Center = map camera center.
- Radius = distance from center to nearest visible map edge.
- Pinch zoom changes radius.
- Pan changes center.
- A visible circle and distance badge make the selected area explicit.

### Q5/Q15 â€” Typed search

**Locked:** keep typed search for city, neighborhood, or address.

Typed search is a **Search area jump**: it recenters the map so the user can tune the final area. It is not a separate final selection mode.

### Q6 â€” Placement

**Locked:** Setup keeps one compact chip that opens a full-screen editor.

Do not render an inline map on Setup. The full-screen editor gives enough room for pan/pinch, search, circle, pins, and commit controls.

### Q7-Q10/Q16 â€” Density preview pins

**Locked:** show **Density preview pins** inside the selected circle.

Pins are non-interactive markers from broad Apple MapKit food/dining search. They help users feel whether the radius is too sparse or too broad. They are not candidate venues, recommendations, venue details, or final options.

Rules:

- Pins appear only inside the selected circle.
- Pins refresh after map movement settles, not continuously during every camera tick.
- Cap around 20 visible pins.
- No venue cards or detail payload on tap.
- No cuisine, quiz, open-now, ranking, or eligibility filters.

### Q11 â€” Commit behavior

**Locked:** explicit commit only.

Map movement does not auto-save. The user taps the editor's primary action to commit the Search area. If the user tries to leave with uncommitted map changes, prompt them to either use the area or discard changes.

### Q12 â€” Setup chip readout

**Locked:** committed chip shows center label plus radius.

Example:

- Main: `Mission Â· San Francisco`
- Sub: `Search area Â· 1.5 mi`

Empty state: `Set search area`.

### Q13 â€” Initial state

**Locked:**

- No committed value + location permission granted: center on current location with a 2.0 mi radius.
- No committed value + permission denied/not granted: open a search-first state.
- Editing an existing Plan: start from saved Search area.
- New Plans do not default to a prior Plan's Search area; they start from current location.

### Q14 â€” Validation

**Locked:**

- `Save for later`: Search area optional.
- `Drop the invite link` / `Start the quiz`: Search area required.
- If launch is tapped with no Search area, open the Search area editor instead of minting the Room.

### Q17/Q19 â€” Radius controls

**Locked:** show bottom-center radius badge above the commit action, outside the circle.

Example: `2.0 MI RADIUS`.

Visible minus/plus step buttons flank the badge for users who cannot comfortably pinch. These controls use the same allowed radius stops and keep map zoom in sync.

### Q18 â€” Editor chrome

**Locked:** minimal full-screen editor chrome:

- close/back
- top search field
- full-screen Apple MapKit map
- floating current-location button
- bottom-center radius badge with minus/plus controls
- bottom primary action: `USE THIS AREA`
- dirty close prompt: `Use this area` / `Discard changes`

### Q20 â€” Component identity

**Locked:** create new `C-28 SearchAreaPicker`.


### Q21 â€” Schema

**Locked:** no schema change for this work.

Persist Search area into the existing center/radius fields (`location_*`, `distance_meters`, `radius_meters`). Rename app/domain language, not database columns, for 0.1.0.
