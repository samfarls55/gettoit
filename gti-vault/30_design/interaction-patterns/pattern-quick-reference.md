# Pattern quick reference

Alphabetical index of every named pattern in this catalog with its trigger condition. Use this when you have a user problem and need to spot the candidate pattern; then jump to the named pattern's full entry for *how to apply* and *audit signals*.

For cognition/behavior principles (Safe Exploration, Habituation, etc.), see [[01-foundations-cognition]].

## A

- **Accordion** — collapsible vertical stack of titled sections, only one open at a time. Use when many related sections share screen real estate and only one is needed at a time. See [[04-layout#Accordion]].
- **Action Panel** — persistent panel grouping the verbs available on the current object. Use when an object has many actions and you don't want them buried in a menu. See [[08-actions#Action Panel]].
- **Alpha/Numeric Scroller** — A-Z (or 0-9) rail beside a long sorted list for jumping to a letter. Use when the user knows the name/number they want and the list is too long to scroll. See [[07-lists#Alpha/Numeric Scroller]].
- **Alternative Views** — same data, multiple presentations (list vs grid vs map). Use when different users or tasks need different presentations of the same content. See [[02-information-architecture#Alternative Views]].
- **Animated Transition** — short motion bridging two states or two screens. Use when a change of context could disorient (zoom, paging, modal open). See [[03-navigation#Animated Transition]].
- **Annotated Scroll Bar** — scroll bar augmented with markers for landmarks/matches. Use when a long scroll has notable points (search hits, errors, sections). See [[03-navigation#Annotated Scroll Bar]].
- **Autocompletion** — type-ahead suggestions filling fields from a known set. Use when the value the user is typing belongs to a finite known set. See [[10-forms#Autocompletion]].

## B

- **Bottom Navigation** — primary nav fixed to bottom of mobile screen. Use when the app has 3-5 top-level sections and users move between them frequently on mobile. See [[06-mobile#Bottom Navigation]].
- **Breadcrumbs** — horizontal trail showing the user's current location in a hierarchy. Use when content lives in a multi-level tree the user navigates by drilling and zooming. See [[03-navigation#Breadcrumbs]].
- **Button Groups** — visually clustered buttons that perform related actions. Use when several actions logically belong together and you want users to scan the group as a unit. See [[08-actions#Button Groups]].

## C

- **Cancelability** — every long operation has a stop/cancel that returns the user to the prior state. Use when an op takes longer than a moment, or could fail. See [[08-actions#Cancelability]].
- **Canvas Plus Palette** — large work area + adjacent tool palette. Use when the surface is a "Make" type (editor, painter, builder). See [[02-information-architecture#Canvas Plus Palette]].
- **Cards** — bounded units of content with a consistent visual frame. Use when each item has multiple kinds of information and the list mixes content types. See [[07-lists#Cards]] and [[06-mobile#Collections and Cards]].
- **Carousel** — horizontally swipeable strip of cards, one focused. Use when items are visually rich and the user is browsing rather than searching. See [[07-lists#Carousel]].
- **Center Stage** — give the most important content the largest visual share of the screen. Use on Focus surfaces. See [[04-layout#Center Stage]].
- **Clear Entry Points** — first-screen tiles for the top 1-3 tasks, prominently rendered. Use when the app supports many things but most users want a few. See [[03-navigation#Clear Entry Points]].
- **Collapsible Panels** — side or stacked panels the user can hide. Use when secondary tools eat real estate during focused work. See [[04-layout#Collapsible Panels]].
- **Collections and Cards** — mobile-tuned list of card units. Use on mobile lists where each item carries multiple fields. See [[06-mobile#Collections and Cards]].
- **Command History** — ordered log of past commands the user has issued. Use when users may want to repeat, undo, or audit prior actions. See [[08-actions#Command History]].

## D

- **Dashboard** — single screen aggregating multiple data widgets and shortcuts. Use when the user's job is monitoring + responding across multiple data streams. See [[02-information-architecture#Dashboard]].
- **Data Brushing** — selecting in one chart highlights the same items in linked charts. Use on data-exploration surfaces with multiple coordinated views. See [[09-complex-data#Data Brushing]].
- **Data Spotlight** — highlight a subset of the data while greying the rest. Use when the user needs to focus on one dimension without losing context. See [[09-complex-data#Data Spotlight]].
- **Datatips** — hover/tap reveals the underlying value of a chart point/cell. Use on any data graphic where the precise number matters. See [[09-complex-data#Datatips]].
- **Deep Links** — every meaningful state of the app has a stable, shareable URL. Use everywhere on the web; required for sharing, bookmarking, search indexing. See [[03-navigation#Deep Links]].
- **Drop-down Chooser** — compact control opening a list/picker on demand. Use for one-of-many selection from a known set. See [[10-forms#Drop-down Chooser]].
- **Dynamic Queries** — live filters that update the result set with each control change. Use when users explore a data set and need to see effects of constraints in real time. See [[09-complex-data#Dynamic Queries]].

## E

- **Error Messages** — concise, in-context messages that name the problem and the fix. Use whenever input validation or system failure can occur. See [[10-forms#Error Messages]].
- **Escape Hatch** — a clear way out of a flow the user wandered into. Use on modal panels, wizards, or any narrow flow. See [[03-navigation#Escape Hatch]].

## F

- **Fat Menus** — large multi-column drop-down nav showing the structure of a section in full. Use when global navigation must expose many destinations without nested menus. See [[03-navigation#Fat Menus]].
- **Feature, Search, and Browse** — landing-page triad: featured content + search box + browseable list. Use on content-heavy sites with diverse user intents. See [[02-information-architecture#Feature, Search, and Browse]].
- **Fill-in-the-Blanks** — natural-language sentences with input slots inline. Use when the input set is small and reads naturally as prose. See [[10-forms#Fill-in-the-Blanks]].
- **Filmstrip** — swipe-paged horizontal sequence of detail screens. Use when comparing or stepping through a small ordered set on mobile. See [[06-mobile#Filmstrip]].
- **Forgiving Format** — accept input in multiple notations and parse the intent. Use for fields where users plausibly type a date/phone/address differently. See [[10-forms#Forgiving Format]].

## G

- **Generous Borders** — large padding around touch targets and between elements on mobile. Use everywhere on mobile to reduce mis-taps. See [[06-mobile#Generous Borders]].
- **Good Defaults and Smart Prefills** — fields pre-filled with the most likely answer based on context. Use whenever a sensible guess can be inferred from context or past behavior. See [[10-forms#Good Defaults and Smart Prefills]].
- **Grid of Equals** — N visually equivalent tiles arranged in a grid. Use when multiple items are co-equal and the user is meant to choose or compare. See [[04-layout#Grid of Equals]].

## H

- **Help Systems** — embedded help layered from tooltips through tours through long-form docs. Use on any non-trivial app. See [[02-information-architecture#Help Systems]].
- **Hover or Pop-Up Tools** — controls appear on hover/long-press over an object. Use when controls would clutter the default view but must be quickly reachable. See [[08-actions#Hover or Pop-Up Tools]].

## I

- **Infinite List** — items load as the user scrolls. Use when feed-like content has no natural end and the user is grazing. See [[06-mobile#Infinite List]].
- **Input Hints** — short cue beside or below a field describing the format/expectation. Use when format isn't obvious from the label. See [[10-forms#Input Hints]].
- **Input Prompt** — placeholder text inside an empty field. Use for short cues only; never replace the label. See [[10-forms#Input Prompt]].

## J

- **Jump to Item** — direct nav to a specific item without scrolling through the list. Use on long alphabetical/sorted lists. See [[07-lists#Jump to Item]].

## L

- **List Builder** — two-pane control for moving items between an "available" and a "chosen" set. Use when the user picks an arbitrary subset from a larger pool. See [[10-forms#List Builder]].
- **List Inlay** — list item expands inline to reveal detail without leaving the list. Use when most of an item's info fits in a row but occasional detail is needed. See [[07-lists#List Inlay]].
- **Loading or Progress Indicators** — visible signal that the system is working, with progress where possible. Use for any wait >300ms; show real progress for >2s waits. See [[06-mobile#Loading or Progress Indicators]] and [[08-actions#Spinners and Loading Indicators]].

## M

- **Macros** — user-recorded sequence of actions playable as one command. Use for tasks users repeat with the same parameters. See [[08-actions#Macros]].
- **Make It Mobile** — design from mobile first, then expand; mobile is not a shrink-down. Use when planning multi-platform delivery. See [[06-mobile#Make It Mobile]].
- **Many Workspaces** — multiple concurrent open documents/tabs/projects. Use when users juggle parallel tasks. See [[02-information-architecture#Many Workspaces]].
- **Media Browser** — grid + filter + detail-pane structure tuned to images/video/audio. Use for any large media library. See [[02-information-architecture#Media Browser]].
- **Menu Page** — entry page whose only content is a list of links to subsections. Use when the app's primary job is to route users somewhere else. See [[03-navigation#Menu Page]].
- **Mobile Direct Access** — first mobile screen acts immediately on device signals (location, camera, time). Use for single-purpose mobile apps. See [[02-information-architecture#Mobile Direct Access]].
- **Modal Panel** — overlay that requires resolution before the user proceeds. Use only when an action genuinely cannot be deferred. See [[03-navigation#Modal Panel]].
- **Module Tabs** — horizontal tabs switching the content of a single content panel. Use when 2-7 related views share a single region. See [[04-layout#Module Tabs]].
- **Movable Panels** — user-rearrangeable panels for expert customization. Use in expert tools where users develop personal layouts. See [[04-layout#Movable Panels]].
- **Multilevel Undo** — undo stack going back many steps, with a redo path. Use on any Make surface. See [[08-actions#Multilevel Undo]].
- **Multi-Y Graph** — single chart with multiple Y-axes for series of different units. Use when series share a time axis but not units. See [[09-complex-data#Multi-Y Graph]].

## N

- **New-Item Row** — explicit row/control in a list that creates a new item. Use when adding items is a frequent action. See [[07-lists#New-Item Row]].

## O

- **One-Window Drilldown** — tap an item, page replaces with its detail. Use on mobile or constrained screens. See [[07-lists#One-Window Drilldown]].

## P

- **Pagination** — explicit page-N-of-M chunking of a long list. Use when scroll exhaustion or jump-to-specific-position matters. See [[07-lists#Pagination]].
- **Password Strength Meter** — live indicator of password robustness as user types. Use on any password field. See [[10-forms#Password Strength Meter]].
- **Preview** — show the result of an action before the user commits. Use whenever the action has visible output the user might want to verify. See [[08-actions#Preview]].
- **Progress Indicator** — show step N of M in a multi-step flow. Use on wizards, checkout, multi-step forms. See [[03-navigation#Progress Indicator]].
- **Prominent "Done" Button or Assumed Next Step** — primary forward action visually dominant. Use at the end of any Do/Wizard step. See [[08-actions#Prominent "Done" Button or Assumed Next Step]].
- **Pyramid (nav)** — sibling navigation between peers under the same parent. Use when users want next/prev among siblings (slide deck, photo set, article). See [[03-navigation#Pyramid]].

## R

- **Richly Connected Apps** — mobile apps that hand off to other apps via OS share, deep links, intents. Use when your app produces or consumes content that lives elsewhere. See [[06-mobile#Richly Connected Apps]].

## S

- **Settings Editor** — grouped, searchable list of preferences with sane defaults and per-item revert. Use for any non-trivial config surface. See [[02-information-architecture#Settings Editor]].
- **Sign-In Tools** — account/auth controls placed where users expect them (top-right web, profile tab mobile). Use whenever auth is required. See [[03-navigation#Sign-In Tools]].
- **Sitemap Footer** — fat global sitemap at the bottom of every page. Use on content sites with deep IA. See [[03-navigation#Sitemap Footer]].
- **Small Multiples** — repeated mini-charts in a grid, one per category, sharing axes. Use to compare distributions across many categories. See [[09-complex-data#Small Multiples]].
- **Smart Menu Items** — menu labels that update to reflect the current target ("Undo Delete" vs "Undo"). Use on action menus. See [[08-actions#Smart Menu Items]].
- **Spinners and Loading Indicators** — see Loading or Progress Indicators above. Spinner = indeterminate wait. See [[08-actions#Spinners and Loading Indicators]].
- **Streams and Feeds** — reverse-chronological scrolling list of cards. Use on social, news, activity. See [[02-information-architecture#Streams and Feeds]].
- **Structured Format** — input split into format-bound subfields (date as MM/DD/YYYY slots). Use when the format is rigid and the user benefits from seeing the structure. See [[10-forms#Structured Format]].

## T

- **Tags** — user- or system-applied labels that aggregate items across categories. Use when items don't fit cleanly in one taxonomy. See [[02-information-architecture#Tags]].
- **Thumbnail Grid** — grid of images, each a clickable item. Use for image-dominant catalogs. See [[07-lists#Thumbnail Grid]].
- **Titled Sections** — horizontal page divisions with explicit section titles. Use to break a long page into scannable chunks. See [[04-layout#Titled Sections]].
- **Touch Tools** — chrome-free direct-manipulation gestures on mobile (tap, swipe, pinch). Use when the UI is direct-manipulation by nature (maps, images, lists). See [[06-mobile#Touch Tools]].
- **Two-Panel Selector or Split View** — list on one side, detail on the other. Use on tablets/desktops with master-detail data. See [[07-lists#Two-Panel Selector or Split View]].

## V

- **Vertical Stack** — primary mobile layout: single column of stacked blocks. Use as the default mobile layout. See [[06-mobile#Vertical Stack]].
- **Visual Framework** — consistent positions for chrome elements across pages. Use on multi-page apps to support [[01-foundations-cognition#P-09. Spatial Memory]]. See [[04-layout#Visual Framework]].

## W

- **Wizard** — linear step-by-step flow with progress indicator. Use only when steps are genuinely sequential and the user must complete them in order. See [[02-information-architecture#Wizard]].
