# bug-17 — Web verdict surface does not conform to locked web-01-invitee-shell §C

## Symptom

The web invitee verdict surface — `VerdictReadOnly` plus the `web/lib/verdict.ts`
shaping helpers — renders a full TB-15-era verdict: venue hero, a meta line, a
fixed "7:00 PM" time badge, the rule text, a row of per-member receipt chips, a
collapsible "what got cut" drawer, and a "Start a new decision" CTA.

The locked design-system spec `design-system/surfaces/web-01-invitee-shell.md`
§C ("Read-only verdict card", status `locked`, 2026-05-21) says the web invitee
verdict card is **plan name + verdict venue only** — and spells out, verbatim,
"No receipts, no per-axis cuts, no rule chip" and "No primary CTA". The surface
does not conform to its own locked spec.

## How this issue was re-scoped (triage + grill, 2026-05-22)

bug-17 was filed by an AFK-run subagent as "web verdict-read path still
references retired v1 typed `votes` columns". A `/triage` + `/grill-with-docs`
session on 2026-05-22 found the original framing wrong on two counts:

- **The reported symptom does not reproduce.** `SessionRoom` already passes
  `votes: []` into `shapeVerdictView`. There is no live read of the dropped
  columns — nothing crashes, nothing renders malformed. The retired-column
  `VoteSummaryRow` type is dead code that exists only to feed receipts §C says
  must not render at all.
- **The suggested fix referenced a ghost.** The original direction — "move
  receipts onto the `verdict_for_room` RPC projection" — is not actionable: no
  `verdict_for_room` RPC exists anywhere in `supabase/`. Only a stale comment
  in `web/lib/mixed-platform.test.ts` claims it does.

Root cause: `VerdictReadOnly.tsx` has not been touched since TB-15 (PR #35).
tb-WF-12 ("read-only verdict card wiring") routed into the component but never
reconciled it to the §C spec locked on 2026-05-21. The retired-column type debt
is one symptom of that whole shaping layer being superseded.

Web/iOS verdict parity is **not** a goal — that was the original issue's false
premise. iOS keeps its full receipt surface; the web invitee card is, by §C, a
far smaller read.

## Grill decisions (2026-05-22)

- **Q1 — scope.** bug-17 becomes a §C-conformance fix for the web verdict
  surface (content + layout), not a `verdict.ts` type patch and not a receipts
  wire-up.
- **Q2 — no-survivor.** §C does not spec the `no_survivor` decided-plan case.
  bug-17 ships a minimal no-survivor variant (plan name + a "No spot fits"
  card, no votes-derived meta line); the §C gap is flagged as a separate
  design-system spec-gap follow-up.
- **Q3 — claim-code mint line.** §C requires the "Getting the app?" mint line
  on the verdict card (a returning invitee of a decided plan lands on §C, not
  Waiting, so a Waiting-only affordance strands them). It is currently wired
  into the web Waiting screen only. bug-17 adds it to the verdict surface.
- **Q4 — live update.** The §C "Live update" reroll cross-fade is out of scope
  — a separate realtime behavior, flagged for its own issue.

#### Agent Brief

**Category:** bug

**Summary:** Conform the web invitee verdict surface to the locked
`web-01-invitee-shell.md` §C spec — plan name + venue only — and add the
§C-required "Getting the app?" claim-code mint line.

**Current behavior:**
The web invitee verdict surface renders a TB-15-era full verdict: venue hero,
meta line, a fixed time badge, rule text, per-member receipt chips, a
collapsible cuts drawer, and a "Start a new decision" CTA. `web/lib/verdict.ts`
carries a `VoteSummaryRow` type and `actionFor` / `survivingHardNeeds` helpers
typed against retired v1 `votes` columns; `SessionRoom` already feeds the
verdict shaper an empty `votes` array, so the receipt machinery never runs on
real data. The "Getting the app?" claim-code mint affordance is wired into the
web Waiting screen only — not the verdict surface.

**Desired behavior:**
The web invitee verdict surface conforms to `web-01-invitee-shell.md` §C:

- *Default (a venue was chosen):* eyebrow, the plan name, and the verdict venue
  in a single card — nothing else. No receipts, no cuts drawer, no rule chip,
  no time badge, no primary CTA.
- *No-survivor (`method: no_survivor`):* the same minimal register — plan name
  plus a "No spot fits" card in the venue slot. No votes-derived hard-needs
  meta line (its only data source, `votes`, is ephemeral and gone by
  decided-time).
- The §C "Getting the app?" claim-code mint line renders below the verdict card
  on both variants, mirroring the affordance already wired into the web Waiting
  screen.
- The dead retired-column shaping layer is removed.

**Key interfaces:**

- The web verdict shaper (`shapeVerdictView` in `web/lib/verdict.ts`) — its
  `VerdictView` result collapses to the §C surface: a place / plan name, a
  no-survivor flag, and nothing receipt- or cut-derived. The `votes` input
  parameter, `VoteSummaryRow`, `actionFor`, `survivingHardNeeds`, the `Receipt`
  / `Cut` types, and the `receipts` / `cuts` fields are removed.
- The web verdict component (`VerdictReadOnly`) — strips the time badge, the
  rule paragraph, the receipts row, the cuts drawer, and the "Start a new
  decision" CTA; renders the §C layout (wordmark, eyebrow, plan name, venue
  card) and the "Getting the app?" mint line below the card.
- `SessionRoom`'s verdict phase — passes an `onMintClaimCode` handler to the
  verdict surface, the same way its Waiting phase already does. The verdict
  fetch no longer needs to load `option_cuts` or `votes` for the view.
- The shared `GettingTheAppAffordance` component — reused as-is on the verdict
  surface.

**Acceptance criteria:**

- [ ] The web verdict surface renders only: eyebrow, plan name, the verdict
      venue card, and the "Getting the app?" mint line — matching
      `web-01-invitee-shell.md` §C.
- [ ] No receipts, no cuts drawer, no rule text, no time badge, and no
      "Start a new decision" CTA appear on the web verdict surface.
- [ ] A `no_survivor` decided plan renders the minimal variant: plan name + a
      "No spot fits" card + the mint line.
- [ ] The "Getting the app?" line mints a claim code from the verdict surface,
      mirroring the web Waiting screen wiring.
- [ ] `VoteSummaryRow`, `actionFor`, `survivingHardNeeds`, and the receipt /
      cut shaping are removed from `web/lib/verdict.ts`; no retired v1 typed
