# GetToIt iOS

Swift + SwiftUI app. iOS 17 minimum (per [ADR 0008](../gti-vault/60_engineering/adr/0008-ios-min-target-17.md)).

## Layout

```
ios/
  project.yml          # XcodeGen spec — source of truth for the Xcode project
  Sources/
    GTITokens.swift    # Generated by design-system/scripts/gen-swift.mjs — do not edit by hand
    App/
      GetToItApp.swift       # SwiftUI App entry point
      RootView.swift         # Walking-skeleton root view (TB-01)
      AuthCoordinator.swift  # Anonymous sign-in (TB-01); extends to Apple link in TB-12
      SupabaseConfig.swift   # Reads SUPABASE_PROJECT_URL/ANON_KEY from Info.plist
      Info.plist
  Tests/
    AnonAuthIntegrationTests.swift  # Hits real Supabase project, checks JWT claims
    GTITokensTests.swift            # Smoke test on the generated tokens
```

The `.xcodeproj` is **not committed** — it is generated from `project.yml`
by [XcodeGen](https://github.com/yonaskolb/XcodeGen) on the macOS CI runner
(and locally if you have it installed).

## Local generation (macOS only)

```sh
brew install xcodegen
cd ios
xcodegen generate
open GetToIt.xcodeproj
```

Inject Supabase credentials from `.env` before opening:

```sh
set -a && source ../.env && set +a
xcodegen generate
```

## CI

The macOS lane in `.github/workflows/ci.yml` runs:

1. `xcodegen generate` — materialize the project from `project.yml`.
2. `xcodebuild test -scheme GetToIt -destination 'platform=iOS Simulator,name=iPhone 15,OS=17.5'`.

`SUPABASE_PROJECT_URL` and `SUPABASE_ANON_KEY` are injected into the build
via `xcodebuild` build-setting overrides from the `gh secret` store.
