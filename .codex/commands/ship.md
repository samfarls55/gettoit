# Ship TestFlight

Ship the active Expo React Native app in `mobile/` to TestFlight by dispatching `.github/workflows/testflight.yml`.

## Arguments

- `ref=<branch|tag|sha>`: code ref to ship. Default: `main`.
- `submit=false` or `build-only`: build without TestFlight submit. Default: submit.
- `notes="..."`: optional TestFlight "What to Test" notes. Default is generated by the workflow.

## Preflight

1. Confirm this is the GetToIt repo and `mobile/` is the active Expo app.
2. Confirm `gh` is installed and authenticated.
3. Check `git status --short`. Warn that uncommitted local changes are not shipped; the workflow checks out the requested GitHub ref.
4. Use these defaults unless arguments override them:
   - `git_ref`: `main`
   - EAS profile: `production`
   - submit: `true`
   - workflow environment: `testflight`
5. Do not continue if `.github/workflows/testflight.yml` is missing.

## Plan

Print the exact shipment plan before dispatching:

- Git ref to ship.
- EAS profile: `production`.
- Submit mode: TestFlight submit or build-only.
- Release notes source: provided notes or workflow default.
- GitHub Environment gate: `testflight`.

Ask the user for explicit confirmation. Require a clear yes before running the command.

## Commands

After confirmation, run from the repo root:

```powershell
.\scripts\ship-testflight.ps1 -Ref "<ref>" -Yes
```

For build-only:

```powershell
.\scripts\ship-testflight.ps1 -Ref "<ref>" -BuildOnly -Yes
```

With release notes:

```powershell
.\scripts\ship-testflight.ps1 -Ref "<ref>" -ReleaseNotes "<notes>" -Yes
```

If the workflow has not landed on `main` yet, pass `-WorkflowRef "<branch>"` for shakedown only.

## Verification

After dispatch:

1. Report the latest run URL from `gh run list --workflow testflight.yml --limit 1`.
2. Remind the user that GitHub Environment `testflight` approval is required before EAS starts.
3. If `-Watch` was requested, run `gh run watch <run-id>`.

## Summary

Report:

```text
Action: dispatched TestFlight workflow
Git ref: <ref>
Submit: true|false
Run URL: <url>
```

## Next Steps

After GitHub approval, inspect the workflow summary for EAS build metadata and the TestFlight submission result.
